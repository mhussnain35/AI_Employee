# Gmail API Setup Guide

This guide walks you through setting up Gmail API access for the AI Employee GmailWatcher.

---

## Overview

The GmailWatcher monitors your Gmail inbox for new, unread, and important emails. When it detects a new email, it creates an action file in the `Needs_Action/` folder for processing.

**Authentication Method:** OAuth2 (secure, token-based)

---

## Step 1: Create Google Cloud Project

1. Go to [Google Cloud Console](https://console.cloud.google.com/)
2. Sign in with your Google account
3. Click **"Select a Project"** → **"New Project"**
4. Name it: `AI Employee` (or any name you prefer)
5. Click **"Create"**

**Time:** ~2 minutes

---

## Step 2: Enable Gmail API

1. In your new project, go to **"APIs & Services"** → **"Library"**
2. Search for **"Gmail API"**
3. Click on **"Gmail API"**
4. Click **"Enable"**

**Time:** ~1 minute

---

## Step 3: Create OAuth2 Credentials

1. Go to **"APIs & Services"** → **"Credentials"**
2. Click **"+ CREATE CREDENTIALS"** → **"OAuth client ID"**
3. If prompted, configure the **OAuth consent screen**:
   - **User Type:** External
   - **App name:** AI Employee
   - **User support email:** Your email
   - **Developer contact:** Your email
   - Click **"Save and Continue"**
   - **Scopes:** Skip this step (we'll use default)
   - **Test users:** Add your Google email address
   - Click **"Save and Continue"**

4. Back to **Create OAuth Client ID**:
   - **Application type:** Desktop app
   - **Name:** AI Employee Gmail
   - Click **"Create"**

5. Download the JSON file by clicking the **Download** icon
6. Rename the file to `credentials.json`

**Time:** ~5 minutes

---

## Step 4: Install Credentials

1. Navigate to your AI Employee project folder
2. Create a `credentials/` folder (if it doesn't exist):
   ```bash
   mkdir credentials
   ```

3. Move the `credentials.json` file:
   ```bash
   mv path/to/downloads/credentials.json credentials/
   ```

**Time:** ~1 minute

---

## Step 5: Authenticate

Run the authentication script:

```bash
python main.py setup-gmail
```

Or directly:

```bash
python scripts/gmail_auth.py
```

**What happens:**
1. A browser window will open
2. Google will ask you to sign in
3. You'll see a warning: "This app isn't verified" (normal for personal projects)
   - Click **"Advanced"** → **"Go to AI Employee (unsafe)"**
4. Grant permissions to read Gmail
5. A `token.json` file will be created in `credentials/`

**Time:** ~3 minutes

---

## Step 6: Verify Authentication

After successful authentication, you should see:

```
✓ Authentication successful!
✓ Successfully connected to Gmail!
  Account: your-email@gmail.com

✓ All checks passed! You can now use GmailWatcher.
```

The `credentials/` folder should now contain:
- `credentials.json` (you downloaded from Google)
- `token.json` (auto-generated by auth script)

---

## Step 7: Start Gmail Watcher

```bash
python main.py watcher-gmail
```

Or start all watchers:

```bash
python main.py watcher-all
```

The watcher will:
- Check Gmail every 2 minutes (by default)
- Look for unread, important emails
- Create action files in `Needs_Action/` folder
- Track processed emails to avoid duplicates

## Troubleshooting

### Error: "credentials.json not found"

**Solution:**
```bash
# Check if file exists
ls credentials/credentials.json  # Mac/Linux
dir credentials\credentials.json  # Windows

# If missing, download again from Google Cloud Console
```

### Error: "Token refresh failed"

**Solution:**
```bash
# Delete old token and re-authenticate
rm credentials/token.json  # Mac/Linux
del credentials\token.json  # Windows

# Re-run authentication
python main.py setup-gmail
```

### Error: "This app isn't verified"

**This is normal** for personal projects. Google shows this warning for apps that haven't gone through their verification process.

**Solution:**
1. Click **"Advanced"**
2. Click **"Go to AI Employee (unsafe)"**
3. Continue with authentication

### GmailWatcher not detecting emails

**Possible causes:**
1. No unread emails
2. All emails marked as not important
3. Token expired

**Solution:**
```bash
# Check token status
python scripts/gmail_auth.py

# Send yourself a test email and mark it important
# Then run watcher again
```

## Configuration Options

### Change Check Interval

Edit the watcher in `main.py`:

```python
# Default: 120 seconds (2 minutes)
gmail_watcher = GmailWatcher(vault_path, check_interval=60)  # 1 minute
```

### Modify Search Query

Edit `watchers/gmail_watcher.py`:

```python
# Default: unread and important
q='is:unread is:important'

# All unread emails
q='is:unread'

# From specific sender
q='from:boss@company.com is:unread'

# With specific label
q='label:Work is:unread'
```

## Security Notes

### Token Storage

- `token.json` contains your access credentials
- It's automatically excluded from Git (`.gitignore`)
- Token auto-refreshes when expired
- Revoke access anytime from [Google Security Settings](https://myaccount.google.com/permissions)

### Scopes

The GmailWatcher requests **read-only** access:
- `https://www.googleapis.com/auth/gmail.readonly`

It **cannot**:
- Send emails
- Delete emails
- Modify emails
- Access your contacts

## Best Practices

1. **Use Labels:** Create a Gmail label "AI Employee" for emails you want processed
2. **Filters:** Set up Gmail filters to auto-mark important emails
3. **Regular Checks:** Run the watcher periodically or keep it running
4. **Token Backup:** Backup `credentials/` folder securely
5. **Monitor Usage:** Check [Google Cloud Dashboard](https://console.cloud.google.com/) for API usage

---

## API Usage Limits

Gmail API has generous limits:
- **1,000,000 units per day** (free)
- Each email read: ~1 unit
- Our watcher uses ~720 units/day (checking every 2 minutes)

**You won't hit limits** with normal usage.
